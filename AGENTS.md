# 作用域

本文件作用于仓库根目录；若子目录存在更近的 `AGENTS.md`，则由更近的文件覆盖其子树内的同类规则。

# 规则启用

只有写入 `AGENTS.md` 的规则才视为仓库协作约束。对话里的临时要求、口头偏好或未落盘说明，不会自动成为持续生效的仓库规则。

# 优先级

1. 对同一路径，距离当前文件最近的 `AGENTS.md` 优先。
2. 下层 `AGENTS.md` 未重写的规则，继续继承上层约束。

# 变更边界

1. 开始修改前，先做与当前任务相关的最小文件和入口盘点。
2. 只改与当前任务直接相关的文件，不顺手扩写功能、重构结构或批量清理风格。
3. 未经确认，不新增模块、workspace 边界、执行入口、注册关系或新的运行模式。
4. 不回滚、覆盖或清理自己未引入的改动，除非用户明确要求。

# 文件边界

1. `AGENTS.md` 只承载协作约束、暂停条件和风险护栏。
2. `README.md` 只承载项目概述、职责边界和阅读索引。
3. `justfile` 只承载稳定执行入口，不混入协作规则或项目背景。

# 暂停条件

1. 当需求边界不清，且存在多种互斥实现时，先暂停并请求确认。
2. 当需要新增模块、root 级入口、workspace 边界或执行面模式时，先暂停并请求确认。
3. 当本地事实与需求冲突，且无法通过最小补丁消解时，先暂停说明冲突点。

# 人工接管

1. 涉及破坏性操作、用户数据外部副作用或不可逆历史修改前，先取得确认。
2. 当实现依赖仓库内无法推断的产品策略、策略边界或运维选择时，升级为人工裁决。

# 工程基线

1. 优先提交最小、可裁决、任务内聚的补丁，不做机会主义重构。
2. 保持现有架构和命令入口稳定、显式，不为局部便利破坏长期分层。
3. 交付前执行最小相关验证；若无法验证，必须说明缺口、原因和风险。
4. 代码、注释、日志和文档保持直接、准确、可验证。
